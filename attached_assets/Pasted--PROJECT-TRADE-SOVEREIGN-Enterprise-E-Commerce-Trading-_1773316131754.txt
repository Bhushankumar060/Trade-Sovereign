# PROJECT: TRADE SOVEREIGN (Enterprise E-Commerce & Trading Platform)
# ROLE: Senior Full-Stack Developer, Cybersecurity Architect, AI Engineer
# GOAL: Build a production-ready, secure marketplace using Firebase, Razorpay, and Google Gemini.
# DESIGN AESTHETIC: Lovable.dev / Emergent (Modern, Glassmorphism, Dark Mode, Smooth Animations).

## 1. TECH STACK REQUIREMENTS
- **Framework:** Next.js 14 (App Router) with TypeScript.
- **Styling:** Tailwind CSS + Framer Motion + Shadcn/UI.
- **Backend & Auth:** Firebase (Auth, Firestore, Storage) + Firebase Admin SDK.
- **Payments:** Razorpay (Checkout, Subscriptions, Webhooks).
- **AI Integration:** Google Gemini API (for Trading Insights & Product Search).
- **Trading:** TradingView Lightweight Charts Widget.
- **Security:** Firebase Security Rules, Razorpay Signature Verification, CSP Headers.

## 2. CYBERSECURITY PROTOCOLS (MANDATORY)
- **Firebase Security:** Implement strict Firestore Security Rules. Users can only read their own data. Admins can write.
- **Payment Security:** Verify Razorpay payment signatures on the server-side API route before confirming order status.
- **AI Security:** Sanitize all inputs sent to Gemini API. Rate limit AI requests to prevent abuse.
- **Authentication:** Use Firebase Auth with Email/Password + Google Sign-in. Implement MFA for Admin accounts.
- **Data Privacy:** Encrypt sensitive user data in Firestore. Do not store payment details (Razorpay handles this).
- **Bot Protection:** Implement Cloudflare Turnstile on login and checkout forms.

## 3. DATABASE SCHEMA (Firebase Firestore)
Structure collections as follows:
- `users`: (uid, email, role [user/admin], loyalty_points, created_at)
- `products`: (id, name, price, category, stock, image_url, is_digital)
- `media`: (id, title, type [music/movie], price, file_url, license_type)
- `orders`: (id, user_id, razorpay_order_id, status, total, created_at)
- `subscriptions`: (id, user_id, plan_type, status, expires_at)
- `rewards`: (id, user_id, points, type, description)
- `trading_analytics`: (id, user_id, symbol, data_json)

## 6. DEBUGGING & FIXING INSTRUCTIONS
- **If Firebase Auth Fails:** Check `firebaseConfig` in `.env`. Ensure CORS is allowed in Firebase Console.
- **If Razorpay Payment Fails:** Check `razorpay_key_id` and `razorpay_key_secret`. Verify Webhook URL in Razorpay Dashboard.
- **If Gemini API Fails:** Check API Key validity. Ensure the server route is protected.
- **If Security Rules Block:** Check Firestore Security Rules. Ensure `request.auth.uid == resource.data.uid` for user data.
- **General:** If any error occurs, log it to the console and display a user-friendly message, do not expose stack traces.
